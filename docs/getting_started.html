Getting Started

Visionary Render has various programmable areas that permit you to optimise your workflow or improve user experience. The pages in this section introduce two of these areas: scripting and plugins.

Scripting

Visionary Render permits you to script behaviour using Lua, which is a high-level programming language that is easy to learn. Visionary Render supports Lua 5.1 (with some extension support for Lua 5.2). For an introduction to the language itself, see the book Programming in Lua, Second Edition by Roberto Ierusalimschy. There are also plenty of free tutorials available online.

The following pages cover scripting:

  • Using the Script Event System

Function Reference

Visionary Render provides various Lua functions for use in the scripting environment. These can be browsed at a glance on the Lua API reference page, or the examples in the How To section can be used to get an idea of some of the things it can be used for.

You can also use the search box at the top of every page in this guide to search for words relating to what you want to do.

Plugins

Plugins permit you to extend the functionality of Visionary Render in a way that persists between different scenes. The Plugin SDK is an extension to the core Visionary Render scripting capabilities. It provides documentation and helper libraries for the purpose of interacting natively with the C API. The SDK examples and helpers are written with C++ in mind, but the C API can be used by any language that is capable of invoking dynamic library functions, as long as it can be bundled up into a binary exporting the required Plugin API functions, loadable by Visionary Render.

Lua Plugins

A Lua plugin is just a specially crafted Lua script that defines a module (so something can require it) with a few simple identifying functions. It is loaded into the Visionary Render scripting environment, meaning that any scripts running on Events or in the Script Console will also have access to the functions that a Lua plugin provides. Lua plugins can be developed with a standard installation of Visionary Render for loading them. There are no external libraries required and the plugin can be developed in any text editor.

The following pages cover Lua plugins:

  • Creating a Lua Plugin

  • Examples

Native Plugins

A native plugin runs compiled code natively as part of the execution of the host application (i.e. Visionary Render). By contrast, a Lua script runs inside the Lua script engine. Native plugins benefit from faster execution. The executable code is compiled into a dynamic library (dll) and is then loaded at runtime by the application plugin manager. A native plugin is expected to export a number of required entry points to allow the host to identify and authorize it access to the VRTree API.
