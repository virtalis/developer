Using the Visionary Render script events system

Introduction to the VR Lua event system and registers

As well as the console window, Visionary Render can execute Lua scripts in response to various scene actions. These are configured using Event nodes in the tree (right click on a node, Create, Event).

This creates an Event node and a Script child. To edit the script, view the properties of the node you added the event to, and find the event you want in the events table (there will only be one if the event was just created).

The application provides a drag and drop interface to auto-generate some basic script actions. These are not covered here.

To edit the script, click the cell containing the word "Script", in the Action column. This will open the script in a new tab in the Script Editor.

Event Types

There are a number of different types of event that can trigger a script. In order to get event specific data into the script, we set a collection of global variables depending on the event type. We call these registers and they are prefixed with two underscores (__).

The table below lists available events, and also which event-specific registers are set. References to "the node" refer to the node to which the event handler is attached. For single nodes that have multiple events responding to the same event, the order in which these events are executed is arbitrary. There is no defined execution order for events on a single node other than the fact that Create events are executed before any other type of event.

Fired when a node is selected. __Other contains the view selection node containing the selection. Note that selecting in the Scenes or Developer views will fire this event twice (once for each view, as the selection is synchronized)

Fired when a subsequent fly-to operation flies somewhere else

Registers

The table below lists the registers and a more detailed description of their purpose.

Register

Type

Description

__Event

vrnode

The event node that was triggered (available in all events)

The name of the gesture that was made

__Hand

boolean

The hand that triggered the event (false = left, true = right)

__KeyCode

number

The value of the key that was pressed. Tracker buttons start at 1000, mouse buttons start at 2000

__KeyState

number

Whether the key in __KeyCode was pressed or released (1 = pressed)

vrnode

string

The name of the pose that was made

__PoseState

boolean

The state of the pose (true = entered, false = exited)

__Script

vrnode

The script node containing the code being executed (available in all events)

__Self

vrnode

The node that the event node is attached to (e.g. an Assembly) - available in all events

__Time

number

The total number of seconds since the application started (available in all events)

number

The time in seconds since the last call to the event

__User

vrnode

The user who triggered the event (available in all events)
