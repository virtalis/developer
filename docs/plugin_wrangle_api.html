Accessing the API

Loading the VRTree library and getting functions to call

The VR API functions are exported from vrtree.dll, which will already be resident in memory by the time your plugin is loaded. Most of these functions require a valid security context (see plugin_license).

A full list of all of the available functions can be found in other areas of this documentation - when searching for entry points please note that all of the named functions are prefixed with __vrexport_ to avoid potential name collisions.

For example, the documented VRCloseNodeHandle function is exported from vrtree.dll as __vrexport_VRCloseNodeHandle.

VR Exchange API functions are exported from vrtree-exchange.dll, which will also be resident in memory by the time your plugin is loaded. Most of these functions also require a valid security context.

Wrangler (via helper libraries)

This SDK provides a wrangler for C / C++ (vrtree-linker) for all of the VR API functions provided by VRTree, as well as fall-back dummy implementations for cases where the version of vrtree.dll differs from the version expected by the plugin.

The dummy implementations provided by the wrangler do little other than prevent the plugin from crashing when calling a null function pointer.

When using the wrangler with C / C++, the VRPLUGIN_LOADVRTREE macro will take care of all of the work. It is usually best to call this during VRPInit.

  • C++
#include <vrtree_api.h>

using namespace vrtree_cpp;

PLUGIN_ENTRY_POINT int VRTREE_APIENTRY VRPInit()
{
  // Query the vrtree DLL for API entry points and load them
  VRPLUGIN_LOADVRTREE;

  HNodeR user = VRGetThisUser();
  // do something with this user node...
  return 0;
}
There is not currently a wrangler for the VR Exchange API. This library must be loaded manually (see below).

Manual Loading

For cases where the wrangler is unsuitable (i.e. because you are not using C/C++, or want to selectively probe for API functions), you should load the DLL using LoadLibrary on Windows. Then, using GetProcAddress, query the functions with the _vrexport prefix. Never assume that a function exists unless you are also checking the API version.

For example, to load the VR Exchange API, you might do the following (error checking omitted for brevity):

  • C / C++
HMODULE hExchangeDLL = LoadLibrary("vrtree-exchange.dll");

typedef uint32_t(*VRXImportFunc)(const char*, HNode, HNode, const char*);
VRXImportFunc importFunc = (VRXImportFunc)GetProcAddress(hExchangeDLL, "__vrexport_VRXImport");

importFunc("path/to/file/to/import", ...);
